# Azure DevOps SDLC � Epics & User Stories

## Business Problem

A digital booking solution required a structured Agile delivery approach to organise business requirements, break work into manageable delivery items, and provide visibility across the development lifecycle.

## Solution

Azure DevOps was used to structure the project backlog and translate business requirements into epics, features, user stories, acceptance criteria, and delivery tasks.

## Agile Delivery Configuration

- Epics and feature breakdown
- User story creation and backlog management
- Gherkin-style acceptance criteria
- Task breakdown and delivery tracking
- Sprint board organisation
- Delivery planning and work item visibility

## Business Analysis Approach

Business requirements were analysed and translated into structured backlog items. User stories were documented from the user perspective, supported by clear acceptance criteria and task-level delivery activities.

## Outcome

The project demonstrates practical understanding of Agile SDLC delivery, requirements management, backlog structuring, user story development, acceptance criteria, and cross-functional delivery tracking.
